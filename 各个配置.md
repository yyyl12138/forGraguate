# 毕设项目：Hyperledger Fabric 2.5 联盟链全景配置

## 1. 项目概述

```md
# 项目名称
Hyperledger Fabric 2.5 联盟链部署（双组织 + 单 Orderer + CouchDB富查询支持）

# 架构目标
构建一个最小可运行的 Fabric 联盟链网络，满足以下约束：
- 双组织（Org1 + Org2）
- 单 Orderer 节点
- 每个组织 1 个 Peer（底层挂载 CouchDB 状态数据库）
- 使用 cryptogen 生成密码学材料
- 使用 Docker 容器化跨机部署
- 使用单 Channel（mychannel）

# 共识与策略
- Orderer 类型：etcdraft（单节点模式）
- 背书策略：OR('Org1MSP.peer','Org2MSP.peer')

# Channel
- Channel 名称：mychannel
- 成员组织：Org1MSP, Org2MSP
```

## 6. 核心配置文件归档

### 6.1 crypto-config.yaml

```yaml
# ---------------------------------------------------------------------------
# "OrdererOrgs" 定义了共识节点所属的组织架构
# ---------------------------------------------------------------------------
OrdererOrgs:
  - Name: Orderer                 # Orderer 组织名称 (对应 OrdererOrg)
    Domain: example.com           # Orderer 组织的根域名
    EnableNodeOUs: true           # 启用节点组织单元(OU)。Fabric 2.x 核心特性，通过证书 OU 区分 client/peer/admin/orderer 身份
    Specs:
      - Hostname: orderer         # 生成的完整域名将为: orderer.example.com
        SANS:                     # 备用名称扩展 (Subject Alternative Names)。多机部署下，防止 TLS 校验因 IP 请求而失败
          - localhost
          - 127.0.0.1
          - 192.168.88.101        # node1 的实际通信 IP
          - node1                 # node1 主机名

# ---------------------------------------------------------------------------
# "PeerOrgs" 定义了参与业务背书与账本存储的节点组织架构
# ---------------------------------------------------------------------------
PeerOrgs:
  - Name: Org1                    # Org1 组织名称
    Domain: org1.example.com      # Org1 组织的根域名
    EnableNodeOUs: true           # 启用节点组织单元(OU)，规范 MSP 身份解析
    Template:
      Count: 1                    # 节点数量。Count: 1 会默认生成名为 peer0 的节点密码学材料 (peer0.org1.example.com)
      SANS:                       # 注入到 Peer 节点 TLS 证书中的备用 IP 和域名
        - localhost
        - 127.0.0.1
        - 192.168.88.102          # node2 的实际通信 IP
        - node2                   # node2 主机名
    Users:
      Count: 1                    # 普通用户数量。cryptogen 默认必定生成 1 个 Admin 用户，Count: 1 表示额外生成 1 个普通 User1 供业务调用测试使用

  - Name: Org2                    # Org2 组织名称
    Domain: org2.example.com      # Org2 组织的根域名
    EnableNodeOUs: true           # 启用节点组织单元(OU)
    Template:
      Count: 1                    # 生成 peer0.org2.example.com
      SANS:                       # 注入到 Peer 节点 TLS 证书中的备用 IP 和域名
        - localhost
        - 127.0.0.1
        - 192.168.88.103          # node3 的实际通信 IP
        - node3                   # node3 主机名
    Users:
      Count: 1                    # 额外生成 1 个普通 User1
```

2. configtx.yaml

```yaml
# ---------------------------------------------------------------------------
# Organizations (组织定义)
# ---------------------------------------------------------------------------
Organizations:
  - &OrdererOrg
    Name: OrdererOrg
    ID: OrdererMSP
    # MSP 目录严格对应 cryptogen 生成的结构
    MSPDir: crypto-config/ordererOrganizations/example.com/msp
    Policies:
      Readers:
        Type: Signature
        Rule: "OR('OrdererMSP.member')"
      Writers:
        Type: Signature
        Rule: "OR('OrdererMSP.member')"
      Admins:
        Type: Signature
        Rule: "OR('OrdererMSP.admin')"
    OrdererEndpoints:
      - orderer.example.com:7050

  - &Org1
    Name: Org1MSP
    ID: Org1MSP
    MSPDir: crypto-config/peerOrganizations/org1.example.com/msp
    Policies:
      Readers:
        Type: Signature
        Rule: "OR('Org1MSP.admin', 'Org1MSP.peer', 'Org1MSP.client')"
      Writers:
        Type: Signature
        Rule: "OR('Org1MSP.admin', 'Org1MSP.client')"
      Admins:
        Type: Signature
        Rule: "OR('Org1MSP.admin')"
      Endorsement:
        Type: Signature
        Rule: "OR('Org1MSP.peer')"
    AnchorPeers:
      # 跨组织通信的锚节点，对应 node2 的服务
      - Host: peer0.org1.example.com
        Port: 7051

  - &Org2
    Name: Org2MSP
    ID: Org2MSP
    MSPDir: crypto-config/peerOrganizations/org2.example.com/msp
    Policies:
      Readers:
        Type: Signature
        Rule: "OR('Org2MSP.admin', 'Org2MSP.peer', 'Org2MSP.client')"
      Writers:
        Type: Signature
        Rule: "OR('Org2MSP.admin', 'Org2MSP.client')"
      Admins:
        Type: Signature
        Rule: "OR('Org2MSP.admin')"
      Endorsement:
        Type: Signature
        Rule: "OR('Org2MSP.peer')"
    AnchorPeers:
      # 跨组织通信的锚节点，对应 node3 的服务
      - Host: peer0.org2.example.com
        Port: 7051

# ---------------------------------------------------------------------------
# Capabilities (能力集配置，Fabric 2.5 需全部启用 V2_0)
# ---------------------------------------------------------------------------
Capabilities:
  Channel: &ChannelCapabilities
    V2_0: true
  Orderer: &OrdererCapabilities
    V2_0: true
  Application: &ApplicationCapabilities
    V2_0: true

# ---------------------------------------------------------------------------
# Application (应用通道默认配置)
# ---------------------------------------------------------------------------
Application: &ApplicationDefaults
  Organizations:
  Policies:
    Readers:
      Type: ImplicitMeta
      Rule: "ANY Readers"
    Writers:
      Type: ImplicitMeta
      Rule: "ANY Writers"
    Admins:
      Type: ImplicitMeta
      Rule: "MAJORITY Admins"
    LifecycleEndorsement:
      Type: ImplicitMeta
      Rule: "MAJORITY Endorsement"
    Endorsement:
      Type: ImplicitMeta
      # 对应需求：满足任意一个组织的背书即可 OR('Org1MSP.peer','Org2MSP.peer')
      Rule: "ANY Endorsement"
  Capabilities:
    <<: *ApplicationCapabilities

# ---------------------------------------------------------------------------
# Orderer (共识节点配置)
# ---------------------------------------------------------------------------
Orderer: &OrdererDefaults
  OrdererType: etcdraft
  Addresses:
    - orderer.example.com:7050
  EtcdRaft:
    Consenters:
      # 单节点 etcdraft 配置
      - Host: orderer.example.com
        Port: 7050
        ClientTLSCert: crypto-config/ordererOrganizations/example.com/orderers/orderer.example.com/tls/server.crt
        ServerTLSCert: crypto-config/ordererOrganizations/example.com/orderers/orderer.example.com/tls/server.crt
  BatchTimeout: 2s
  BatchSize:
    MaxMessageCount: 10
    AbsoluteMaxBytes: 99 MB
    PreferredMaxBytes: 512 KB
  Organizations:
  Policies:
    Readers:
      Type: ImplicitMeta
      Rule: "ANY Readers"
    Writers:
      Type: ImplicitMeta
      Rule: "ANY Writers"
    Admins:
      Type: ImplicitMeta
      Rule: "MAJORITY Admins"
    BlockValidation:
      Type: ImplicitMeta
      Rule: "ANY Writers"
  Capabilities:
    <<: *OrdererCapabilities

# ---------------------------------------------------------------------------
# Channel (通道默认配置)
# ---------------------------------------------------------------------------
Channel: &ChannelDefaults
  Policies:
    Readers:
      Type: ImplicitMeta
      Rule: "ANY Readers"
    Writers:
      Type: ImplicitMeta
      Rule: "ANY Writers"
    Admins:
      Type: ImplicitMeta
      Rule: "MAJORITY Admins"
  Capabilities:
    <<: *ChannelCapabilities

# ---------------------------------------------------------------------------
# Profiles (配置模板，用于生成创世区块和通道交易)
# ---------------------------------------------------------------------------
Profiles:
  # 生成系统创世区块 (genesis.block)
  OrdererGenesis:
    <<: *ChannelDefaults
    Orderer:
      <<: *OrdererDefaults
      Organizations:
        - *OrdererOrg
    Consortiums:
      SampleConsortium:
        Organizations:
          - *Org1
          - *Org2

  # 生成通道配置交易 (channel.tx)
  ChannelProfile:
    Consortium: SampleConsortium
    <<: *ChannelDefaults
    Application:
      <<: *ApplicationDefaults
      Organizations:
        - *Org1
        - *Org2
```
